## نطاق العمل (Scope of Work)

[قسّم النطاق إلى وحدات عمل واضحة، مرتبطة بالميزات في `features-map.md`.]

### 1. تحليل النظام الحالي

- تحليل الكود المصدري للتطبيق الحالي (مثلاً: Flutter App).
- تحليل الـ Backend الحالي (مثلاً: ASP.NET Core APIs) وقاعدة البيانات.
- تحديد نقاط التكامل لإضافة وحدات [الميزة الأساسية] (مثل: الولاء، الاشتراكات، التقارير).

### 2. تطوير الخدمات الخلفية (Backend Services)

- إنشاء جداول وقواعد بيانات لإدارة البيانات الخاصة بالميزة (مثل: `LoyaltyTransactions`, `UserPoints`).
- تطوير واجهات برمجية (APIs) تشمل على سبيل المثال:
  - `EarnPoints API`
  - `RedeemPoints API`
  - `UserPointsBalance API`
- تأمين الـ APIs باستخدام آليات مثل: `JWT`, `HTTPS`, Role-based Authorization.

### 3. تعديل واجهات المستخدم (Frontend & Mobile App)

- تحديث واجهة التطبيق (مثلاً: `Flutter`) لعرض البيانات الجديدة (رصيد نقاط، حالة اشتراك، ...).
- إضافة خيارات تفعيل الميزة (مثل: استخدام النقاط في شاشة Checkout).
- تحديث واجهة لوحة التحكم (مثلاً: `Angular Admin Panel`) لإدارة السياسات والإعدادات.

### 4. التقارير والإدارة (Admin Panel Integration)

- إضافة شاشات تقارير تظهر:
  - إجمالي البيانات/النقاط لكل مستخدم أو لكل فترة.
  - مؤشرات أداء الميزة (Usage, Redemption Rate, Active Users) حسب طبيعة المشروع.
- تمكين الفريق من تعديل سياسات الميزة (نسب التحويل، شروط الاستحقاق، حدود الاستخدام، ...).

(حدّث أسماء الوحدات وفقًا للميزة/النظام الفعلي.)
